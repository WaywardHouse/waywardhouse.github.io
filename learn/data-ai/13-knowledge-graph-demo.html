15  Knowledge Graph Demo

The goal of this demo is to show a wayfinding approach that does not depend on hardwired URLs inside prose.

Instead, pages declare a small amount of structured metadata:

  • a stable id (page identity)
  • concepts (what this page teaches/uses)
  • prereq_concepts (what this page assumes)

A build script then generates a small knowledge graph (nodes + edges) and a resolver (page id → output href). Those artefacts can be used for:

  • computed “Related” panels
  • back-links (“where else is this concept used?”)
  • a site-wide concept index
  • graph views
